## 1. 系統概述

### 1.1 專案背景

> 📝 *簡述此系統的背景與開發動機（2–3 句）*

本系統為 IM2002 資料庫管理課程期末作業，目標為建構一個虛構雙網絡交通運營商「TransitFlow」的後端資料庫系統，並透過 AI 聊天助理讓使用者能以自然語言查詢交通資訊、訂票、退票等服務。

### 1.2 系統目標

- [ ] 建立可運行的 PostgreSQL 關聯式資料庫（含完整 Schema）
- [ ] 建立 Neo4j 圖形資料庫（含地鐵 / 國鐵路網）
- [ ] 整合 pgvector 向量資料庫（RAG 政策文件查詢）
- [ ] 實作所有 Agent 工具對應的 Query 函式
- [ ] 能透過 Gradio 介面正常與 AI 互動

### 1.3 系統範疇（Scope）

**包含：**
- 地鐵（Metro）與國鐵（National Rail）資訊查詢
- 國鐵訂票 / 取消訂票
- 乘客帳號登入 / 註冊
- 政策文件語意搜尋（RAG）
- 路線規劃與換乘建議
- 延誤影響分析

**不包含：**
- 即時班次更新
- 線上金流串接
- 行動裝置 APP

## 3. 資料庫規格

### 3.1 PostgreSQL 關聯式 Schema

> 📝 *說明各資料表設計決策*

#### 3.1.1 資料表清單

| 資料表名稱 | 對應 JSON | 主鍵 | 說明 |
|-----------|-----------|------|------|
| `metro_stations` | metro_stations.json | `station_id` | 地鐵站基本資料 |
| `metro_lines` 或 `station_lines` | metro_stations.json | — | 站別所屬路線（正規化或 ARRAY） |
| `metro_schedules` | metro_schedules.json | `schedule_id` | 地鐵班表 |
| `metro_schedule_stops` | metro_schedules.json | — | 班表停靠站（正規化） |
| `national_rail_stations` | national_rail_stations.json | `station_id` | 國鐵站基本資料 |
| `national_rail_schedules` | national_rail_schedules.json | `schedule_id` | 國鐵班表 |
| `national_rail_stops` | national_rail_schedules.json | — | 國鐵停靠站與時刻 |
| `national_rail_fares` | national_rail_schedules.json | — | 國鐵票價區間 |
| `seat_layouts` | national_rail_seat_layouts.json | `seat_id` | 國鐵座位配置 |
| `users` | registered_users.json | `user_id` | 乘客帳號 |
| `bookings` | bookings.json | `booking_id` | 國鐵訂票記錄 |
| `metro_travel_history` | metro_travel_history.json | `trip_id` | 地鐵乘車歷史 |
| `payments` | payments.json | `payment_id` | 付款記錄 |
| `feedback` | feedback.json | `feedback_id` | 乘客評價 |
| `policy_documents` | （向量表）| `id` | RAG 政策文件 |

#### 3.1.3 主要資料表 ER 圖（文字版）

```
users ──1:N──► bookings ──N:1──► national_rail_schedules
                   │                      │
                   └──1:1──► payments     └──1:N──► national_rail_stops
                                          └──1:N──► national_rail_fares

national_rail_schedules ──N:M──► seat_layouts
bookings ──N:1──► seat_layouts

metro_stations ──N:M──► metro_schedules（透過 metro_schedule_stops）
users ──1:N──► metro_travel_history
```

### 3.2 Neo4j 圖形 Schema

#### 3.2.1 節點（Node）規格

| 標籤 | 屬性 | 說明 |
|------|------|------|
| `:MetroStation` | `station_id`, `name`, `lines[]`, `zone` | 地鐵站 |
| `:NationalRailStation` | `station_id`, `name`, `city` | 國鐵站 |

#### 3.2.2 關係（Relationship）規格

| 關係類型 | 起點 → 終點 | 屬性 | 說明 |
|---------|------------|------|------|
| `:METRO_LINK` | MetroStation → MetroStation | `line`, `travel_time_min` | 地鐵相鄰站連線 |
| `:RAIL_LINK` | NationalRailStation → NationalRailStation | `line`, `travel_time_min` | 國鐵相鄰站連線 |
| `:INTERCHANGE_TO` | MetroStation → NationalRailStation | `walk_time_min` | 換乘連接 |

#### 3.2.3 設計決策

| 議題 | 決策 |
|------|------|
| 地鐵 / 國鐵站是否合併為一個 Label | 分開（`:MetroStation` + `:NationalRailStation`） |
| 邊上是否存 `fare_per_stop` | __________ |
| 路線方向（有向 / 無向圖） | __________ |

### 3.3 pgvector 向量 Schema
